# 定时发布任务配置

## 快速开始

### 1. 设置系统 Cron（推荐）

在服务器上执行 `crontab -e`，添加以下行：

```
* * * * * php /path/to/seo-publisher/cron/publish.php >> /path/to/seo-publisher/cron/publish.log 2>&1
```

这会每分钟检查一次是否有到期需要发布的文章。

### 2. 查看日志

```bash
tail -f /path/to/seo-publisher/cron/publish.log
```

### 3. 手动触发

如果不想配置cron，也可以在"生成文章"页面点击"立即处理队列"按钮手动触发发布。

## 工作流程

1. 在"生成文章"页面生成文章（状态：待生成 → 已生成）
2. 点击"开始发布"按钮，系统根据发布设置计算每篇文章的发布时间
3. 文章进入"待发布"队列（状态：scheduled），显示计划发布时间
4. Cron任务或手动处理会在到达发布时间时自动发布文章

## 发布设置说明

在"站点管理 → 发布设置"中可以配置：

- **发布时间**：每天固定的时间点发布（如 08:00,12:00,18:00）
- **发布间隔**：每篇文章之间的间隔时间（分钟）
- **随机间隔**：勾选后间隔时间会随机化，更自然
- **每日最多发布**：限制每天发布数量

## 常见问题

**Q: 必须开着浏览器吗？**
A: 不需要。配置了系统cron后，后台会自动执行，无需打开浏览器。

**Q: 会消耗网站服务器资源吗？**
A: 每次执行只处理到期的文章，执行完立即退出，资源消耗极低。

**Q: 如何停止自动发布？**
A: 删除crontab规则即可，或者在文章列表中将待发布文章的状态重置。
